# 🧱 1. `stack.py` — реализация стека

Файл содержит класс `Stack`, реализующий стек на основе списка Python.

### Основные методы:

- `push(c)` — добавить элемент в стек
- `pop()` — удалить и вернуть верхний элемент
- `top()` — получить верхний элемент без удаления

```python
class Stack:
    def __init__(self):
        self.array = []

    def push(self, c):
        self.array.append(c)

    def pop(self):
        return self.array.pop()

    def top(self):
        return self.array[len(self.array) - 1]
```

📌 Это базовая структура, на которой строится весь проект.

# ⚙️ 2. `compf.py` — компилятор формул

Класс `Compf` преобразует арифметическое выражение в **постфиксную (обратную польскую) запись**.

### 📌 Что делает компилятор

Пример:
```
a+b      → a b +
a*(b+c)  → a b c + *
```

---

## 🧠 Основные идеи

- Используется стек для хранения операций
- Результат формируется в списке `data`
- Поддерживаются:
  - операции `+ - * /`
  - скобки
  - переменные `a-z`

---

## 🔑 Ключевые методы

### `compile(str)`

```python
def compile(self, str):
    self.data.clear()
    for c in "(" + str + ")":
        self.process_symbol(c)
    return " ".join(self.data)
```

📌 Оборачивает выражение в скобки и обрабатывает посимвольно

---

### `process_symbol(c)`

Определяет тип символа:

- `(` → кладётся в стек
- `)` → выгружаются операции
- оператор → учитывается приоритет
- иначе → переменная

---

### `process_suspended_operators(c)`

Выгружает операции из стека, если они должны выполниться раньше

---

### `priority(c)`

```python
return 1 if (c == "+" or c == "-") else 2
```

📌 `*` и `/` имеют более высокий приоритет

---

### `is_precedes(a, b)`

Определяет, нужно ли выполнять операцию из стека раньше текущей

---

## 📌 Вывод

`Compf` **не вычисляет выражение**, а только переводит его в удобную форму для вычисления.

---

# 🧮 3. `calc.py` — калькулятор

Класс `Calc` наследуется от `Compf` и **вычисляет выражение**.

---

## 🔄 Основная идея

- Использует тот же алгоритм разбора, что и `Compf`
- Но вместо строки формирует результат

---

## ➕ Отличия от `Compf`

### Разрешённые символы:

```python
SYMBOLS = re.compile("[0-9]")
```

📌 Только цифры

---

### Дополнительный стек:

```python
self.r = Stack()
```

📌 Стек для чисел

---

### Переопределённые методы

#### `compile`

```python
def compile(self, str):
    Compf.compile(self, str)
    return self.r.top()
```

---

#### `process_value`

```python
self.r.push(int(c))
```

---

#### `process_oper`

```python
second, first = self.r.pop(), self.r.pop()
self.r.push({"+": add, "-": sub, "*": mul, "/": truediv}[c](first, second))
```

---

## 📌 Вывод

- `Compf` → переводит выражение  
- `Calc` → вычисляет его

# 🧪 4. Тесты (`pytest`)

Тесты проверяют корректность работы программы.

---

## 📦 `test_stack.py`

Проверяет стек.

### Проверки:

- стек пуст после создания
- `push` и `pop` работают
- ошибка при `top()` у пустого стека

---

## 🔤 `test_compf.py`

Проверяет компиляцию формул.

### Примеры:

```python
assert self.c.compile("a+b") == "a b +"
assert self.c.compile("a*(b/c)") == "a b c / *"
```

📌 Проверяется:

- порядок операций
- скобки
- сложные выражения

---

## 🔢 `test_calc.py`

Проверяет вычисления.

### Проверки:

- сложение, вычитание, умножение, деление
- деление на ноль
- сложные выражения

### Пример:

```python
assert self.c.compile('1+2') == 3
assert self.c.compile('1/2') == approx(0.5)
```

---

## ⚠️ Особенности тестов

- `raises` — проверка ошибок
- `approx` — сравнение float
- `eval()` — сравнение с Python
